Default of Credit Card Clients Dataset¶

  • 이 데이터 세트에는 2005 년 4 월부터 2005 년 9 월까지 대만의 신용 카드 고객의 기본 지불, 인구 통계 학적 요인, 신용 데이터, 지불 내역 및 청구서에 대한 정보가 포함되어 있습니다.

  • 24,000개의 샘플과 25개의 속성이 있습니다.

  • LIMIT_BAL : NT 달러(신 타이완 달러)로 표시된 크레딧 금액 (개인 및 가족 / 보충 크레딧 포함)
  • SEX : 성별 (1 = 남성, 2 = 여성)
  • EDUCATION : (1 = 대학원, 2 = 대학, 3 = 고등학교, 4 = 기타, 5 = 알 수 없음, 6 = 알 수 없음)
  • MARRIAGE : 결혼 상태 (1 = 기혼, 2 = 미혼, 3 = 기타)
  • AGE : 나이
  • PAY_0 ~ PAY_6 : n 개월 전 상환 상태 (PAY_0 = 지난달 ... PAY_6 = 6 개월 전) (라벨 : -1 = 정당한 지불, 1 = 한 달 지불 지연, 2 = 2 개월 지불 지연,. .. 8 = 8 개월 지급 지연, 9 = 9 개월 이상 지급 지연)
  • BILL_AMT1 ~ BILL_AMT6 : n 개월 전 청구서 금액 (BILL_AMT1 = last_month .. BILL_AMT6 = 6 개월 전)
  • PAY_AMT1 ~ PAY_AMT6 : n 개월 전 결제 금액 (BILL_AMT1 = last_month .. BILL_AMT6 = 6 개월 전)
  • default : 기본 결제 (1 = 예, 0 = 아니오) [TARGET Value]

설정이 성공적으로 실행되면 몇 가지 중요한 정보가 포함 된 정보 그리드를 인쇄합니다. 대부분의 정보는 setup () 실행시 생성되는 전처리 파이프 라인과 관련이 있습니다. 이러한 기능의 대부분은이 자습서에서 다루지 않지만이 단계에서 유의해야 할 몇 가지 중요한 사항은 다음과 같습니다.

  • session_id : 나중에 재현할 수 있도록 모든 함수에서 시드로 배포 된 pseudo-random 숫자입니다. session_id가 전달되지 않으면 모든 함수에 배포되는 난수가 자동으로 생성됩니다. 이 실험에서는 나중에 재현 할 수 있도록 session_id를 123으로 설정합니다.

  • Target Type : 바이너리 또는 다중 클래스. 타겟 유형이 자동으로 감지되고 표시됩니다. 이진 또는 다중 클래스 문제에 대한 실험 수행 방법에는 차이가 없습니다. 모든 기능은 동일합니다.

  • Label Encoded : Target 변수가 1 또는 0이 아닌 문자열 유형 (예 : 'Yes'또는 'No') 인 경우 자동으로 레이블을 1과 0으로 인코딩하고 다음에 대한 매핑 (0 : 아니요, 1 : 예)을 표시합니다. 참고. 이 실험에서는 대상 변수가 숫자 유형이므로 레이블 인코딩이 필요하지 않습니다.

  • Original Data : 데이터 셋의 원본 모양을 표시합니다. 이 실험에서 (22800, 24)는 22,800 개의 샘플과 대상 컬럼을 포함한 24 개의 피처를 의미합니다.

  • Missing Values : 원본 데이터에 결 측값이있는 경우 True로 표시됩니다. 이 실험에서는 데이터 세트에 결 측값이 없습니다.

  • Numeric Features : 숫자로 추론되는 특성의 수입니다. 이 데이터 세트에서 24 개 기능 중 14 개는 숫자로 추론됩니다.

  • Categorical Features : 범주 형으로 추론 된 특징의 수입니다. 이 데이터 세트에서 24 개 기능 중 9 개는 범주 형으로 추론됩니다.

  • Transformed Train Set : 변형 된 훈련 세트의 모양을 표시합니다. 변환 된 기차 세트에 대해 (22800, 24)의 원래 모양이 (15959, 91)로 변환되고 범주 형 인코딩으로 인해 기능 수가 24 개에서 91 개로 증가했습니다.

  • Transformed Test Set : 변형 된 테스트 / 홀드 아웃 세트의 모양을 표시합니다. 테스트 / 홀드 아웃 세트에는 6841 개의 샘플이 있습니다. 이 분할은 설정에서 train_size 매개 변수를 사용하여 변경할 수있는 기본값 70/30을 기반으로합니다.

create_model() 함수를 사용하여 모델을 만들면 기본 하이퍼 파라미터를 사용하여 모델을 학습시킵니다.

하이퍼 파라미터를 조정하기 위해 tune_model() 함수가 사용됩니다.

이 기능은 미리 정의 된 검색 공간에서 Random Grid Search를 사용하여 모델의 하이퍼 파라미터를 자동으로 조정합니다.

출력은 정확도, AUC, 재현율, 정밀도, F1, Kappa 및 MCC를 fold하여 최상의 모델에 대해 보여주는 점수 그리드를 인쇄합니다.
